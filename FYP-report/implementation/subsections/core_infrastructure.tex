\section{Technical Foundations}
MERIT has undergone several optimisations to ensure its reliability and scalability.

A significant scalability challenge for MERIT was the \textbf{N+1 query bottleneck}.
When retrieving $N$ candidates, the backend was making $N$ subsequent database calls.
This inefficiency was noticeable when processing large candidate pools.
In order to resolve this, we leveraged PostgreSQL's capability for relational embeddings
via the Supabase client. This resulted in the backend transitioning from $O(N)$ to $O(1)$
query complexity, ensuring that our dashboard remains responsive, even under
heavy load.

Another key issue, this time regarding reliability, was present in the
\textit{Data Extraction and Job Configuration} stage, where we deal with the inherent noise in
unstructured data. MERIT leverages a global Flask middleware to intercept exceptions across the
configuration and fusion pipelines. Here, errors are categorised as either recoverable or
critical errors. This is supplemented by strict JSON schema validation at the data ingestion
boundary, in order to ensure that only verified signals enter the \textit{Multi-Source Fusion}
engine, thereby maintaining an explainable audit trail.
