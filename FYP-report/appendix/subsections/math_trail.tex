\chapter{Mathematical Audit Trail}
\label{appendix:math_trail}

This appendix serves as a complete visual breakdown of what a typical MERIT~\cite{merit_repo} metric calculation looks like
for a given candidate. In this case, we consider a programming language metric, Python, this is an extensible
metric derived from a job description (not a hardcoded baseline metric). Here, we show the recruiter the
complete logic behind how the strength of each data source is calculated, and the weights that are applied to 
each source when later performing Bayesian Evidence Fusion. This trail also accounts for heuristic decay,
batch normalisation and also the display of Shapley Values to inform the recruiter whether our score 
was driven by demonstrated evidence, or claimed evidence.
Phase~0--1 shows per-source strength and GitHub-only skill decay (not applied to self-reported timelines).
Phase~2 updates Beta parameters from signal strength and trust, then computes Shapley attributions.
Phase~3 reports the fused expected score, variance, and a High/Medium/Low confidence band.
